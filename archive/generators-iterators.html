Lazy Evaluation and Custom Iteration

Let's explore Python's generators and iterators through TDD. We'll build a Fibonacci generator and a custom iterator.

Step 1: Test Fibonacci Generator (Red Phase)

Start with a test for a Fibonacci generator:

generators_test.py
def test_fibonacci_generator(): fib = fibonacci(5) result = list(fib) assert result == [0, 1, 1, 2, 3]

Run the test: pytest generators_test.py -v

Expected result: The test fails because the fibonacci function doesn't exist yet.

Step 2: Implement Fibonacci Generator (Green Phase)

Write the minimal code to make the test pass:

generators.py
def fibonacci(n): a, b = 0, 1 for _ in range(n): yield a a, b = b, a + b

Run the test again: pytest generators_test.py -v

Expected result: The test now passes! üéâ

Step 5: Test Custom Iterator (Red Phase)

Run the test: pytest generators_test.py -v

Step 6: Implement Custom Iterator (Green Phase)

Add the custom iterator implementation:

generators.py
def fibonacci(n): a, b = 0, 1 for _ in range(n): yield a a, b = b, a + b def square_generator(n): for i in range(n): yield i * i class Countdown: def __init__(self, start): self.start = start def __iter__(self): return self def __next__(self): if self.start <= 0: raise StopIteration current = self.start self.start -= 1 return current

Run the test again: pytest generators_test.py -v

Expected result: All tests now pass! üéâ

What We Learned

  • Generators: Functions that yield values instead of returning them
  • Lazy evaluation: Values are generated on-demand, not all at once
  • Memory efficiency: Generators use less memory than lists
  • Custom iterators: Classes that implement __iter__ and __next__
  • StopIteration: Signal that iteration is complete
