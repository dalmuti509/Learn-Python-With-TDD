Your First Python Program with TDD

Let's start with the classic "Hello, world" program, but we'll do it the TDD way!

Step 1: Write Your First Test (Red Phase)

Start by writing a test that describes what you want your function to do. This test will fail initially because the function doesn't exist yet.

hello_world_test.py
def test_hello_returns_hello_world(): result = hello() assert result == "Hello, world"

Run the test: pytest hello_world_test.py -v

Expected result: The test fails because the hello() function doesn't exist yet.

Step 2: Write Minimal Implementation (Green Phase)

Now write the minimal code to make the test pass. Don't over-engineer it!

hello_world.py
def hello(): return "Hello, world"

Run the test again: pytest hello_world_test.py -v

Step 3: Refactor (Optional)

Since this is a simple function, there's not much to refactor. But in more complex code, this is where you'd improve the code while keeping the tests green.
