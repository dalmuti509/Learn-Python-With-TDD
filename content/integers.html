Working with Numbers and Basic Math

Let's explore Python's integer operations through TDD.

Step 1: Test Addition (Red Phase)

Start with the simplest operation - addition. Write a test for what you want:

calculator_test.py
def test_add(): calc = Calculator() result = calc.add(2, 3) assert result == 5

Run the test: pytest calculator_test.py -v

Expected result: The test fails because the Calculator class doesn't exist yet.

Step 2: Implement Addition (Green Phase)

Write the minimal code to make the test pass:

calculator.py
class Calculator: def add(self, a, b): return a + b

Run the test again: pytest calculator_test.py -v

Step 3: Test Subtraction (Red Phase)

Now add a test for subtraction:

calculator_test.py
def test_add(): calc = Calculator() result = calc.add(2, 3) assert result == 5 def test_subtract(): calc = Calculator() result = calc.subtract(5, 3) assert result == 2

Run the test: pytest calculator_test.py -v

Expected result: The first test passes, but the second fails because subtract doesn't exist yet.

Step 4: Implement Subtraction (Green Phase)

Add the minimal implementation for subtraction:

calculator.py
class Calculator: def add(self, a, b): return a + b def subtract(self, a, b): return a - b

Run the test again: pytest calculator_test.py -v

Expected result: Both tests now pass! ðŸŽ‰

Step 5: Continue the Cycle

You would continue this pattern for multiplication, division, and any other features. Each time:

  1. Red: Write a failing test for the next feature
  2. Green: Write minimal code to make it pass
  3. Refactor: Improve the code while keeping tests green

Key TDD Principles

  • One test at a time: Focus on one small piece of functionality
  • Minimal implementation: Write just enough code to pass the test
  • Incremental progress: Build features one by one
  • Confidence: Each test gives you confidence in your code
