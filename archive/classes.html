Classes and Objects in Python

Let's explore object-oriented programming in Python through TDD. We'll build a simple banking system.

Step 1: Test Account Creation (Red Phase)

Start with a test for creating a basic account:

account_test.py
def test_account_creation(): account = Account("John", 1000) assert account.owner == "John" assert account.balance == 1000

Run the test: pytest account_test.py -v

Expected result: The test fails because the Account class doesn't exist yet.

Step 2: Implement Account Class (Green Phase)

Write the minimal code to make the test pass:

account.py
class Account: def __init__(self, owner, balance): self.owner = owner self.balance = balance

Run the test again: pytest account_test.py -v

Expected result: The test now passes! üéâ

Step 3: Test Deposit Method (Red Phase)

Now add a test for depositing money:

account_test.py
def test_account_creation(): account = Account("John", 1000) assert account.owner == "John" assert account.balance == 1000 def test_deposit(): account = Account("John", 1000) account.deposit(500) assert account.balance == 1500

Run the test: pytest account_test.py -v

Expected result: The first test passes, but the second fails because deposit method doesn't exist yet.

Step 4: Implement Deposit Method (Green Phase)

Add the minimal implementation for deposit:

account.py
class Account: def __init__(self, owner, balance): self.owner = owner self.balance = balance def deposit(self, amount): self.balance += amount

Run the test again: pytest account_test.py -v

Expected result: Both tests now pass! üéâ

Step 5: Test Withdrawal Method (Red Phase)

Add a test for withdrawing money:

account_test.py
def test_account_creation(): account = Account("John", 1000) assert account.owner == "John" assert account.balance == 1000 def test_deposit(): account = Account("John", 1000) account.deposit(500) assert account.balance == 1500 def test_withdraw(): account = Account("John", 1000) account.withdraw(300) assert account.balance == 700

Run the test: pytest account_test.py -v

Expected result: The first two tests pass, but the third fails because withdraw method doesn't exist yet.

Step 6: Implement Withdrawal Method (Green Phase)

Add the implementation for withdrawal:

account.py
class Account: def __init__(self, owner, balance): self.owner = owner self.balance = balance def deposit(self, amount): self.balance += amount def withdraw(self, amount): self.balance -= amount

Run the test again: pytest account_test.py -v

Expected result: All tests now pass! üéâ

What We Learned

  • Class definition: Using class keyword to define objects
  • Constructor: __init__ method for object initialization
  • Instance methods: Functions that operate on object instances
  • Object state: Using instance variables to store data
  • TDD with classes: Building object-oriented code incrementally
