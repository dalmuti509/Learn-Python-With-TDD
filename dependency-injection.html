Dependency Injection Patterns

Let's explore dependency injection through TDD. We'll build a notification system that can send messages via different channels.

Step 1: Test Email Notification (Red Phase)

Start with a test for sending an email notification:

notification_test.py
def test_email_notification(): notifier = EmailNotifier() service = NotificationService(notifier) result = service.send("Hello", "user@example.com") assert result == "Email sent: Hello to user@example.com"

Run the test: pytest notification_test.py -v

Expected result: The test fails because the classes don't exist yet.

Step 2: Implement Email Notification (Green Phase)

Write the minimal code to make the test pass:

notification.py
class EmailNotifier: def send(self, message, recipient): return f"Email sent: {message} to {recipient}" class NotificationService: def __init__(self, notifier): self.notifier = notifier def send(self, message, recipient): return self.notifier.send(message, recipient)

Run the test again: pytest notification_test.py -v

Expected result: The test now passes! üéâ

Expected result: The first test passes, but the second fails because SMSNotifier doesn't exist yet.

Step 5: Test Mock Notification (Red Phase)

Add a test using a mock notifier to verify the service behavior:

notification_test.py
def test_email_notification(): notifier = EmailNotifier() service = NotificationService(notifier) result = service.send("Hello", "user@example.com") assert result == "Email sent: Hello to user@example.com" def test_sms_notification(): notifier = SMSNotifier() service = NotificationService(notifier) result = service.send("Hello", "+1234567890") assert result == "SMS sent: Hello to +1234567890" def test_mock_notification(): mock_notifier = MockNotifier() service = NotificationService(mock_notifier) result = service.send("Test", "test@example.com") assert result == "Mock sent: Test to test@example.com" assert mock_notifier.send_called

Run the test: pytest notification_test.py -v

Expected result: The first two tests pass, but the third fails because MockNotifier doesn't exist yet.

Step 6: Implement Mock Notifier (Green Phase)

Add the mock notifier for testing:

notification.py
class EmailNotifier: def send(self, message, recipient): return f"Email sent: {message} to {recipient}" class SMSNotifier: def send(self, message, recipient): return f"SMS sent: {message} to {recipient}" class MockNotifier: def __init__(self): self.send_called = False def send(self, message, recipient): self.send_called = True return f"Mock sent: {message} to {recipient}" class NotificationService: def __init__(self, notifier): self.notifier = notifier def send(self, message, recipient): return self.notifier.send(message, recipient)

Run the test again: pytest notification_test.py -v

Expected result: All tests now pass! üéâ

What We Learned

  • Dependency injection: Passing dependencies to classes instead of creating them internally
  • Interface segregation: Using common interfaces for different implementations
  • Testability: Easy to test with mock objects
  • Flexibility: Easy to swap implementations
  • Inversion of control: Dependencies are provided from outside
