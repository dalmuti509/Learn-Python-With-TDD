Dependency Injection Patterns

Let's explore dependency injection through TDD. We'll build a notification system that can send messages via different channels.

Step 1: Test Email Notification (Red Phase)

Start with a test for sending an email notification:

notification_test.py
def test_email_notification(): notifier = EmailNotifier() service = NotificationService(notifier) result = service.send("Hello", "user@example.com") assert result == "Email sent: Hello to user@example.com"

Run the test: pytest notification_test.py -v

Expected result: The test fails because the classes don't exist yet.

Step 2: Implement Email Notification (Green Phase)

Write the minimal code to make the test pass:

notification.py
class EmailNotifier: def send(self, message, recipient): return f"Email sent: {message} to {recipient}" class NotificationService: def __init__(self, notifier): self.notifier = notifier def send(self, message, recipient): return self.notifier.send(message, recipient)

Run the test again: pytest notification_test.py -v

Step 3: Test SMS Notification (Red Phase)

Now add a test for SMS notifications using the same service:

notification_test.py
def test_email_notification(): notifier = EmailNotifier() service = NotificationService(notifier) result = service.send("Hello", "user@example.com") assert result == "Email sent: Hello to user@example.com" def test_sms_notification(): notifier = SMSNotifier() service = NotificationService(notifier) result = service.send("Hello", "+1234567890") assert result == "SMS sent: Hello to +1234567890"

Run the test: pytest notification_test.py -v

Expected result: The first test passes, but the second fails because SMSNotifier doesn't exist yet.

Step 4: Implement SMS Notification (Green Phase)

Add the SMS notifier implementation:

notification.py
class EmailNotifier: def send(self, message, recipient): return f"Email sent: {message} to {recipient}" class SMSNotifier: def send(self, message, recipient): return f"SMS sent: {message} to {recipient}" class NotificationService: def __init__(self, notifier): self.notifier = notifier def send(self, message, recipient): return self.notifier.send(message, recipient)

Run the test again: pytest notification_test.py -v

Expected result: Both tests now pass! ðŸŽ‰

What We Learned

  • Dependency injection: Passing dependencies through constructor
  • Interface consistency: Different implementations with same interface
  • Loose coupling: Service doesn't depend on specific notifier types
  • Testability: Easy to test with different implementations
