Resource Management with `with` Statements

Let's explore Python's context managers through TDD. We'll build a file manager and a timer context manager.

Step 1: Test File Context Manager (Red Phase)

Start with a test for a file context manager:

context_managers_test.py
def test_file_context_manager(): with FileManager("test.txt", "w") as f: f.write("Hello, World!") with FileManager("test.txt", "r") as f: content = f.read() assert content == "Hello, World!"

Run the test: pytest context_managers_test.py -v

Expected result: The test fails because the FileManager class doesn't exist yet.

Step 2: Implement File Context Manager (Green Phase)

Write the minimal code to make the test pass:

context_managers.py
class FileManager: def __init__(self, filename, mode): self.filename = filename self.mode = mode self.file = None def __enter__(self): self.file = open(self.filename, self.mode) return self.file def __exit__(self, exc_type, exc_val, exc_tb): if self.file: self.file.close()

Run the test again: pytest context_managers_test.py -v

Expected result: The test now passes! üéâ

Step 3: Test Timer Context Manager (Red Phase)

Now add a test for a timer context manager:

context_managers_test.py
def test_file_context_manager(): with FileManager("test.txt", "w") as f: f.write("Hello, World!") with FileManager("test.txt", "r") as f: content = f.read() assert content == "Hello, World!" def test_timer_context_manager(): with Timer() as t: import time time.sleep(0.1) assert t.elapsed > 0

Step 4: Implement Timer Context Manager (Green Phase)

Add the timer context manager implementation:

context_managers.py
import time class FileManager: def __init__(self, filename, mode): self.filename = filename self.mode = mode self.file = None def __enter__(self): self.file = open(self.filename, self.mode) return self.file def __exit__(self, exc_type, exc_val, exc_tb): if self.file: self.file.close() class Timer: def __enter__(self): self.start = time.time() return self def __exit__(self, exc_type, exc_val, exc_tb): self.elapsed = time.time() - self.start

Run the test again: pytest context_managers_test.py -v

Expected result: Both tests now pass! üéâ

Step 5: Test Context Manager Decorator (Red Phase)

Add a test for a context manager using the @contextmanager decorator:

context_managers_test.py
def test_file_context_manager(): with FileManager("test.txt", "w") as f: f.write("Hello, World!") with FileManager("test.txt", "r") as f: content = f.read() assert content == "Hello, World!" def test_timer_context_manager(): with Timer() as t: import time time.sleep(0.1) assert t.elapsed > 0 def test_temp_file_context_manager(): with temp_file("test.txt") as filename: with open(filename, "w") as f: f.write("Temporary content") assert open(filename).read() == "Temporary content"

Run the test: pytest context_managers_test.py -v

Expected result: The first two tests pass, but the third fails because temp_file function doesn't exist yet.

Step 6: Implement Context Manager Decorator (Green Phase)

Add the context manager using the decorator approach:

context_managers.py
import time import tempfile import os from contextlib import contextmanager class FileManager: def __init__(self, filename, mode): self.filename = filename self.mode = mode self.file = None def __enter__(self): self.file = open(self.filename, self.mode) return self.file def __exit__(self, exc_type, exc_val, exc_tb): if self.file: self.file.close() class Timer: def __enter__(self): self.start = time.time() return self def __exit__(self, exc_type, exc_val, exc_tb): self.elapsed = time.time() - self.start @contextmanager def temp_file(suffix=""): with tempfile.NamedTemporaryFile(delete=False, suffix=suffix) as f: temp_name = f.name try: yield temp_name finally: os.unlink(temp_name)

Run the test again: pytest context_managers_test.py -v

Expected result: All tests now pass! üéâ

What We Learned

  • Context managers: Objects that manage resources with with statements
  • __enter__ and __exit__: Methods that define context manager behavior
  • Resource cleanup: Automatic cleanup when exiting the context
  • @contextmanager decorator: Creating context managers with generators
  • Exception handling: Context managers can handle exceptions in __exit__
